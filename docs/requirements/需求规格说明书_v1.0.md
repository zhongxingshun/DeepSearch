## 1. 项目概述

### 1.1 项目背景
企业积累了大量历史文档资料（约 1TB），目前存储在 NAS 上，查找效率低下。需要构建一个支持全文检索的文档搜索系统，提升知识资产的利用效率。

### 1.2 项目目标
构建一个针对 **1TB 规模** 存量文档及新增文档的深度检索平台。系统通过将 NAS 存储的资料迁移至本地高性能服务器，利用 **Meilisearch** 搜索引擎与 **Kreuzberg** 解析引擎，实现对 PDF、Word、PPT、图片等多种格式内容的秒级全文检索、权限管理及在线下载。

### 1.3 项目范围

**包含范围 (In Scope)**:
- [x] 多格式文档的全文检索
- [x] 文件上传与自动索引
- [x] 用户权限管理
- [x] 文件下载功能

**不包含范围 (Out of Scope)**:
- [ ] 文档在线编辑
- [ ] 文档版本管理
- [ ] 多语言 OCR（仅支持中英文）
- [ ] 移动端 APP

### 2.2 深度搜索 (M-001) 🔴 P0

#### F-001: 多格式文档检索
**优先级**: P0  
**用户故事**: 作为用户，我希望能够搜索各种格式的文档内容，以便快速找到需要的资料。

**功能描述**:
- 支持 PDF（原生/扫描件）、Word (.doc/.docx)、PPT (.ppt/.pptx)、Excel (.xls/.xlsx) 
- 支持图片格式 (JPG/PNG) 的 OCR 文字识别
- 实现毫秒级的关键词匹配及模糊搜索

**验收标准 (AC)**:
- [ ] AC1: 用户输入关键词，系统在 200ms 内返回搜索结果
- [ ] AC2: 搜索结果正确匹配 PDF、Word、PPT、Excel 文档内容
- [ ] AC3: 扫描件 PDF 和图片能通过 OCR 识别后被搜索到
- [ ] AC4: 支持模糊搜索（允许 1-2 个字符的拼写容错）

#### F-002: 内容溯源与高亮
**优先级**: P0  
**用户故事**: 作为用户，我希望看到关键词在文档中的上下文，以便判断是否是我需要的文档。

**功能描述**:
- 搜索结果展示关键词所在的文本片段（Snippet）
- 关键词高亮显示
- 显示文档基本信息（文件名、类型、大小、修改时间）

**验收标准 (AC)**:
- [ ] AC1: 每条搜索结果显示包含关键词的 150 字上下文片段
- [ ] AC2: 关键词在片段中高亮显示
- [ ] AC3: 显示文件名、文件类型、文件大小、最后修改时间

### 2.3 文件管理与自动化 (M-002) 🔴 P0

#### F-003: 后台增量索引
**优先级**: P0  
**用户故事**: 作为系统，我需要能够自动扫描和索引新增文件，保持索引库最新。

**功能描述**:
- 支持"静默处理"模式，后台自动搬运并解析文件
- 不影响前端用户使用
- 支持断点续做

**验收标准 (AC)**:
- [ ] AC1: 系统每 10 分钟自动扫描新增文件
- [ ] AC2: 新文件在 5 分钟内完成索引（单个文件 < 100MB）
- [ ] AC3: 索引任务中断后，重启可自动恢复继续

#### F-004: 手动上传与索引
**优先级**: P1  
**用户故事**: 作为管理员，我希望能够手动上传文件并触发索引。

**功能描述**:
- Web 界面上传文件
- 上传后自动触发 Kreuzberg 解析
- 解析完成后更新 Meilisearch 索引

**验收标准 (AC)**:
- [ ] AC1: 管理员可通过 Web 界面上传单个文件（最大 500MB）
- [ ] AC2: 支持批量上传（最多 20 个文件）
- [ ] AC3: 上传完成后 3 分钟内完成索引更新
- [ ] AC4: 显示上传和索引进度

#### F-005: 文件下载
**优先级**: P0  
**用户故事**: 作为用户，我希望能够下载搜索到的原始文件。

**功能描述**:
- 搜索结果关联物理文件路径
- 点击下载按钮获取原文件

**验收标准 (AC)**:
- [ ] AC1: 每条搜索结果显示下载按钮
- [ ] AC2: 点击下载按钮可获取原始文件
- [ ] AC3: 下载速度 ≥ 10MB/s（局域网环境）

## 4. 技术架构设计

### 4.1 软件栈 (Tech Stack)

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| **搜索引擎** | Meilisearch | 负责索引存储与极速检索 |
| **解析引擎** | Kreuzberg | 负责多格式提取及内置 OCR 处理 |
| **后端框架** | FastAPI (Python) | 负责 API 逻辑、权限校验及任务调度 |
| **任务队列** | Celery + Redis | 负责 1TB 资料的异步、长周期解析任务 |
| **数据库** | PostgreSQL | 存储用户信息、文件元数据、任务状态 |
| **前端界面** | Vue.js 3 | 负责搜索交互及管理后台 |
| **容器化** | Docker + Docker Compose | 统一部署和管理 |
