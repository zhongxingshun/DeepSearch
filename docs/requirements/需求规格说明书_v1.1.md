## 1. 项目概述

### 1.1 项目背景
企业积累了大量历史文档资料（约 1TB），目前存储在 NAS 上，查找效率低下。需要构建一个支持全文检索的文档搜索系统，提升知识资产的利用效率。

### 1.2 项目目标
构建一个针对 **1TB 规模** 存量文档及新增文档的深度检索平台。系统通过将 NAS 存储的资料迁移至本地高性能服务器，利用 **Meilisearch** 搜索引擎与 **Kreuzberg** 解析引擎，实现对 PDF、Word、PPT、图片等多种格式内容的秒级全文检索、权限管理及在线下载。

### 1.3 项目范围

**包含范围 (In Scope)**:
- [x] 多格式文档的全文检索
- [x] 文件上传与自动索引
- [x] 用户权限管理
- [x] 文件下载功能
- [x] 搜索历史功能 *(v1.1 新增)*

**不包含范围 (Out of Scope)**:
- [ ] 文档在线编辑
- [ ] 文档版本管理
- [ ] 多语言 OCR（仅支持中英文）
- [ ] 移动端 APP

### 2.2 深度搜索 (M-001) 🔴 P0

#### F-001: 多格式文档检索
**优先级**: P0  
**用户故事**: 作为用户，我希望能够搜索各种格式的文档内容，以便快速找到需要的资料。

**功能描述**:
- 支持 PDF（原生/扫描件）、Word (.doc/.docx)、PPT (.ppt/.pptx)、Excel (.xls/.xlsx) 
- 支持图片格式 (JPG/PNG) 的 OCR 文字识别
- 实现毫秒级的关键词匹配及模糊搜索
- **Excel 文件索引全部 Sheet 内容** *(v1.1 新增)*

**验收标准 (AC)**:
- [ ] AC1: 用户输入关键词，系统在 200ms 内返回搜索结果
- [ ] AC2: 搜索结果正确匹配 PDF、Word、PPT、Excel 文档内容
- [ ] AC3: 扫描件 PDF 和图片能通过 OCR 识别后被搜索到
- [ ] AC4: 支持模糊搜索（允许 1-2 个字符的拼写容错）
- [ ] **AC5**: 搜索结果默认每页显示 20 条，支持翻页加载 *(v1.1 新增)*
- [ ] **AC6**: 单次搜索最大返回 1000 条结果，超出部分提示用户缩小搜索范围 *(v1.1 新增)*
- [ ] **AC7**: Excel 文件的所有 Sheet 内容均可被搜索到 *(v1.1 新增)*

#### F-002: 内容溯源与高亮
**优先级**: P0

#### F-004: 手动上传与索引
**优先级**: P1

- **支持文件去重检测** *(v1.1 新增)*

**验收标准 (AC)**:
- [ ] AC1: 管理员可通过 Web 界面上传单个文件（最大 500MB）
- [ ] AC2: 支持批量上传（最多 20 个文件）
- [ ] AC3: 上传完成后 3 分钟内完成索引更新
- [ ] AC4: 显示上传和索引进度
- [ ] **AC5**: 系统采用 MD5 哈希检测重复文件，重复文件跳过索引并提示用户 *(v1.1 新增)*
- [ ] **AC6**: 单文件解析超时时间 10 分钟，超时后标记为失败 *(v1.1 新增)*
- [ ] **AC7**: 解析失败的任务自动重试 3 次 *(v1.1 新增)*
- [ ] **AC8**: 最终失败的任务记录到失败列表，并在管理后台显示 *(v1.1 新增)*

**任务队列规格** *(v1.1 新增)*:
| 参数 | 值 |
|------|-----|
| 队列最大长度 | 1000 个任务 |
| 手动上传优先级 | HIGH |
| 后台扫描优先级 | LOW |
| 队列满时策略 | 拒绝新的 LOW 优先级任务，HIGH 优先级任务正常入队 |

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 目标值 | 测试条件 | 优先级 |
|------|--------|---------|--------|
| 搜索响应时间 (P95) | < 200ms | **50 并发用户** *(v1.1 明确)* | P0 |
| 文件下载速度 | ≥ 10MB/s | 局域网环境 | P0 |
| 索引处理速度 | ≥ 30GB/天 | 单节点 | P1 |
| 并发用户数 | ≥ 50 | - | P1 |
| 单文件上传限制 | 500MB | - | P1 |

**性能降级策略** *(v1.1 新增)*:
| 场景 | 处理方式 |
|------|---------|
| 并发 > 50 用户 | 新请求进入等待队列，返回 429 状态码，提示"系统繁忙，请稍后重试" |
| 搜索响应 > 500ms | 触发告警通知管理员 |
| 任务队列 > 1000 | 拒绝新的 LOW 优先级任务 |

### 3.2 可用性需求

| 指标 | 目标值 |
|------|--------|
| 系统可用性 | 99% (工作时间) |
| 索引任务断点续做 | 支持 |
| 故障恢复时间 (RTO) | < 1小时 |
| **恢复点目标 (RPO)** | **≤ 1小时** *(v1.1 新增)* |

**备份策略** *(v1.1 新增)*:
| 备份对象 | 备份频率 | 保留期限 | 说明 |
|---------|---------|---------|------|
| PostgreSQL 数据库 | 每日全量 02:00 + 每小时增量 | 7 天 | 使用 pg_dump + WAL 归档 |
| Meilisearch 索引 | 每日全量 03:00 | 3 天 | 支持从原文件重建索引 |
| 原始文件 | RAID 镜像 + 每周全量备份 | 30 天 | 存储在独立 HDD |
| 系统配置 | 每日全量 01:00 | 7 天 | 配置文件和环境变量 |

## 4. 技术架构设计

### 4.1 软件栈 (Tech Stack)

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| **搜索引擎** | Meilisearch | 负责索引存储与极速检索 |
| **解析引擎** | Kreuzberg | 负责多格式提取及内置 OCR 处理 |
| **后端框架** | FastAPI (Python) | 负责 API 逻辑、权限校验及任务调度 |
| **任务队列** | Celery + Redis | 负责 1TB 资料的异步、长周期解析任务 |

| **前端界面** | Vue.js 3 | 负责搜索交互及管理后台 |
| **容器化** | Docker + Docker Compose | 统一部署和管理 |

## 5. 硬件配置指标 (预算控制：¥3,000)

为了保证 1TB 资料索引完成后的流畅度，硬件需满足以下"低主频、大内存、高 IO"的特性：

| 组件 | 推荐指标/型号 | 预估费用 |
|------|--------------|---------| 
| **CPU** | Intel Xeon E5-2680 v4 (14 核 28 线程) | ¥200 |
| **内存** | **64GB DDR4 ECC** (用于索引缓存) | ¥900 |
| **索引盘** | **1TB NVMe SSD** (必须为全新，存放数据库) | ¥650 |
| **仓库盘** | 4TB 企业级 HDD (存放 1TB 原始资料) | ¥350 |
| **准系统** | HP Z440 或 Dell T5810 (含电源主板) | ¥700 |
| **合计** |  | **约 ¥3,000** |

**索引容量估算** *(v1.1 新增)*:
| 项目 | 估算值 | 说明 |
|------|-------|------|
| 原始文件大小 | 1 TB | 存储在 HDD |
| 预计索引大小 | 100-300 GB | 通常为原文件的 10%-30% |
| 数据库大小 | ~10 GB | 元数据和日志 |
| NVMe SSD 剩余空间 | 600-800 GB | 满足扩展至 2TB 需求 |

> ✅ **结论**: 1TB NVMe SSD 完全满足当前需求，并预留充足扩展空间

### 6.5 数据迁移策略 *(v1.1 新增)*

**迁移概述**:
- 数据源：NAS 存储（1TB 历史文档）
- 目标：本地 HDD（4TB 企业级硬盘）
- 预计迁移时间：3-5 天（取决于网络带宽）

**迁移方式**:
| 阶段 | 方式 | 说明 |
|------|------|------|
| 初始迁移 | rsync 全量同步 | 首次迁移全部文件 |
| 增量同步 | rsync --update | 每日增量同步新增/修改文件 |
| 索引同步 | 后台 Celery 任务 | 检测新文件并建立索引 |

**数据校验机制**:
| 校验项 | 方法 | 执行时机 |
|-------|------|---------|
| 文件完整性 | MD5 哈希对比 | 每个文件迁移完成后 |
| 文件数量 | 源/目标计数对比 | 每日迁移完成后 |
| 目录结构 | 目录树对比 | 初始迁移完成后 |

**迁移期间业务影响**:
- 迁移期间 NAS 访问速度可能下降 10%-20%
- 建议在非工作时间（晚间/周末）进行大批量迁移
- 迁移过程支持暂停/恢复，不影响已迁移文件

**回滚方案**:
- 原始文件保留在 NAS 不删除，直到验证迁移完成且系统稳定运行 30 天
- 如发现数据问题，可从 NAS 重新同步

## 7. 假设与依赖

### 7.1 假设条件
- 假设 NAS 网络传输带宽 ≥ 100Mbps
- 假设大部分 PDF 为可识别文本或高清扫描件
- 假设用户数量 < 100 人
- 假设文档主要为中英文内容

### 7.2 外部依赖

| 依赖项 | 类型 | 状态 | 风险 | 验证计划 *(v1.1 新增)* |
|--------|------|------|------|---------|

| Meilisearch 开源版 | 开源 | ✅ 可用 | 低 | - |
| Kreuzberg 库 | 开源 | ✅ 可用 | 中 | 见下方验证计划 |
| Tesseract OCR | 开源 | ✅ 可用 | 低 | - |

**Kreuzberg 验证计划** *(v1.1 新增)*:

| 验证项目 | 验证内容 | 通过标准 | 验证时间 |
|---------|---------|---------|---------|
| 大文件性能 | 100MB/200MB/500MB PDF 解析 | 解析时间 < 10 分钟 | 第 5 天 |
| 格式兼容性 | 各格式样本文件解析 | 测试集 95% 成功率 | 第 7 天 |
| 内存占用 | 大文件解析内存峰值 | 峰值 < 8GB | 第 5 天 |
| 并发稳定性 | 10 并发解析任务 | 无崩溃/死锁 | 第 10 天 |

**格式兼容性测试清单**:
| 格式 | 测试用例数 | 重点测试场景 |
