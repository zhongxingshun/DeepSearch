## 1. 概述

### 1.1 设计目标

基于需求规格说明书 v1.1，设计一个支持 1TB 规模文档的企业级全文检索系统，实现：

1. **高性能检索**: 50 并发用户下 P95 响应时间 < 200ms
2. **多格式支持**: PDF/Word/PPT/Excel/图片的全文索引与检索
3. **可靠性保障**: 99% 可用性，RPO ≤ 1小时，RTO < 1小时
4. **安全合规**: RBAC 权限控制、审计日志、密码策略

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| 模块化 | 各组件职责单一，松耦合设计 |
| 可扩展 | 支持 2TB+ 数据扩展，新格式解析器扩展 |
| 高可用 | 任务断点续做，故障快速恢复 |
| 安全优先 | 认证鉴权、日志审计、数据加密 |

- 需求规格说明书 v1.1 (SRS)
- 需求评审记录 v2.0

### 4.2 表结构设计

#### 4.2.1 users 表

```sql
CREATE TABLE users (
    id              SERIAL PRIMARY KEY,
    username        VARCHAR(50) UNIQUE NOT NULL,
    password_hash   VARCHAR(255) NOT NULL,
    role            VARCHAR(20) NOT NULL DEFAULT 'user',  -- 'admin' | 'user'
    is_active       BOOLEAN DEFAULT TRUE,
    failed_attempts INTEGER DEFAULT 0,
    locked_until    TIMESTAMP,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_users_username ON users(username);
```

#### 4.2.2 files 表

```sql
CREATE TABLE files (
    id              SERIAL PRIMARY KEY,
    filename        VARCHAR(255) NOT NULL,
    file_path       VARCHAR(1000) NOT NULL,
    uploaded_by     INTEGER REFERENCES users(id),
    file_size       BIGINT NOT NULL,
    file_type       VARCHAR(50) NOT NULL,
    md5_hash        VARCHAR(32) UNIQUE,
    index_status    VARCHAR(20) DEFAULT 'pending',  -- pending|processing|completed|failed
    meilisearch_id  VARCHAR(100),
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_files_md5 ON files(md5_hash);
CREATE INDEX idx_files_status ON files(index_status);
CREATE INDEX idx_files_type ON files(file_type);
```

#### 4.2.3 tasks 表

```sql
CREATE TABLE tasks (
    id              SERIAL PRIMARY KEY,
    file_id         INTEGER REFERENCES files(id),
    task_type       VARCHAR(50) NOT NULL,  -- 'parse' | 'index' | 'scan'
    priority        VARCHAR(10) DEFAULT 'low',  -- 'high' | 'low'
    status          VARCHAR(20) DEFAULT 'pending',
    retry_count     INTEGER DEFAULT 0,
    error_message   TEXT,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    started_at      TIMESTAMP,
    completed_at    TIMESTAMP
);

CREATE INDEX idx_tasks_status ON tasks(status);
CREATE INDEX idx_tasks_priority ON tasks(priority, created_at);
```

#### 4.2.4 search_history 表

```sql
CREATE TABLE search_history (
    id              SERIAL PRIMARY KEY,
    user_id         INTEGER REFERENCES users(id) ON DELETE CASCADE,
    keyword         VARCHAR(500) NOT NULL,
    result_count    INTEGER DEFAULT 0,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_history_user ON search_history(user_id, created_at DESC);
```

#### 4.2.5 audit_logs 表

```sql
CREATE TABLE audit_logs (
    id              SERIAL PRIMARY KEY,
    user_id         INTEGER REFERENCES users(id),
    action_type     VARCHAR(50) NOT NULL,  -- login|logout|download|upload|admin_action
    target_type     VARCHAR(50),
    target_id       VARCHAR(100),
    ip_address      VARCHAR(45),
    details         JSONB,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_audit_user ON audit_logs(user_id, created_at DESC);
CREATE INDEX idx_audit_action ON audit_logs(action_type, created_at DESC);

-- 分区表，按月分区提高查询效率
```

## 7. 性能设计

### 7.1 性能目标

| 指标 | 目标值 | 实现策略 |
|------|--------|---------|
| 搜索响应 P95 | < 200ms | Meilisearch 原生性能 + SSD 存储 |
| 并发用户 | ≥ 50 | Nginx 限流 + 连接池 |
| 下载速度 | ≥ 10MB/s | 直接文件流传输 |
| 索引速度 | ≥ 30GB/天 | Celery 多 Worker 并行 |

### 7.2 优化策略

#### 7.2.1 搜索优化

- Meilisearch 索引存储在 NVMe SSD
- 合理设置 searchableAttributes，避免全字段搜索
- 使用 pagination 限制单次返回数量
