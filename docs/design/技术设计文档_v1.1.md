## 1. 概述

### 1.1 设计目标

基于需求规格说明书 v1.1，设计一个支持 1TB 规模文档的企业级全文检索系统，实现：

1. **高性能检索**: 50 并发用户下 P95 响应时间 < 200ms
2. **多格式支持**: PDF/Word/PPT/Excel/图片的全文索引与检索
3. **可靠性保障**: 99% 可用性，RPO ≤ 1小时，RTO < 1小时
4. **安全合规**: RBAC 权限控制、审计日志、密码策略

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| 模块化 | 各组件职责单一，松耦合设计 |
| 可扩展 | 支持 2TB+ 数据扩展，新格式解析器扩展 |
| 高可用 | 任务断点续做，故障快速恢复 |
| 安全优先 | 认证鉴权、日志审计、数据加密 |

#### 7.4.1 Celery Beat 配置

```python
# celery_config.py
from celery.schedules import crontab

    'pg-incremental-backup': {

        'options': {'queue': 'maintenance'}

    # PostgreSQL 全量备份 - 每日 02:00
    'pg-full-backup': {

    'cleanup-audit-logs': {
        'task': 'app.tasks.cleanup_task.cleanup_audit_logs',

    # 搜索历史清理 - 每日 04:30
    'cleanup-search-history': {
```

#### 7.4.3 备份任务实现

```python
# app/tasks/backup_task.py
from celery import shared_task
import subprocess
from datetime import datetime

@shared_task(bind=True, max_retries=3)
def pg_full_backup(self):
    """PostgreSQL 全量备份"""
    backup_dir = "/data/backups/postgres"
    filename = f"pg_full_{datetime.now().strftime('%Y%m%d_%H%M%S')}.sql.gz"
    
    cmd = f"pg_dump -h postgres -U $DB_USER $DB_NAME | gzip > {backup_dir}/{filename}"
    subprocess.run(cmd, shell=True, check=True)
    
    # 清理超过 7 天的备份
    cleanup_old_backups(backup_dir, days=7)
    
    return {"status": "success", "filename": filename}

@shared_task(bind=True, max_retries=3)
def meili_snapshot(self):
    """Meilisearch 索引快照"""
    import requests
    
    response = requests.post(
        f"http://meilisearch:7700/snapshots",
        headers={"Authorization": f"Bearer {MEILI_MASTER_KEY}"}
    )
    response.raise_for_status()
    
    # 清理超过 3 天的快照
    cleanup_old_snapshots(days=3)
    
    return {"status": "success", "task_uid": response.json()["taskUid"]}
```

### 7.5 缓存设计 *(v1.1 新增)*

#### 7.5.1 缓存使用场景

| 缓存类型 | Key 格式 | TTL | 说明 |
|---------|---------|-----|------|
| 会话缓存 | `session:{token_jti}` | 24h | JWT Token 黑名单（登出时添加） |
| 限流计数器 | `ratelimit:{ip}:{endpoint}` | 1min | 接口限流计数 |
| 用户信息 | `user:{user_id}` | 5min | 当前用户信息缓存 |
| 系统配置 | `config:system` | 10min | 系统配置缓存 |

#### 7.5.2 缓存策略说明

**不缓存搜索结果的原因**:
- Meilisearch 原生响应已足够快（~50ms）
- 搜索结果个性化程度高，缓存命中率低
- 新文档索引后需立即可搜索，缓存会导致数据不一致

#### 7.5.3 缓存实现

```python
# app/core/cache.py
import redis
import json
from functools import wraps
from app.config import settings

redis_client = redis.Redis(
    host=settings.REDIS_HOST,
    port=settings.REDIS_PORT,
    db=0,
    decode_responses=True
)

def cache_user(ttl=300):
    """用户信息缓存装饰器"""
    def decorator(func):
        @wraps(func)
        async def wrapper(user_id: int, *args, **kwargs):
            cache_key = f"user:{user_id}"
            
            # 尝试从缓存获取
            cached = redis_client.get(cache_key)
            if cached:
                return json.loads(cached)
            
            # 缓存未命中，查询数据库
            result = await func(user_id, *args, **kwargs)
            
            # 写入缓存
            redis_client.setex(cache_key, ttl, json.dumps(result))
            
            return result
        return wrapper
    return decorator

class RateLimiter:
    """接口限流器"""
    
    def __init__(self, max_requests: int = 100, window: int = 60):
        self.max_requests = max_requests
        self.window = window
    
    def is_allowed(self, key: str) -> bool:
        """检查是否允许请求"""
        current = redis_client.incr(key)
        if current == 1:
            redis_client.expire(key, self.window)
        return current <= self.max_requests
```

**文件存储分桶策略** *(v1.1 新增)*:

| 配置项 | 值 | 说明 |
|-------|-----|------|
| 分桶方式 | MD5 前 2 位 | 共 256 个子目录 (00-ff) |
| 文件命名 | `{md5}.{ext}` | 使用完整 MD5 作为文件名 |
| 存储路径 | `/data/files/{md5[0:2]}/{md5}.{ext}` | 完整存储路径 |
| 优点 | 分布均匀 | 每个目录约存储 4000 个文件 (1TB/256) |

```python
# 文件路径生成示例
def get_file_path(md5_hash: str, extension: str) -> str:
    bucket = md5_hash[:2]  # 取前两位作为桶名
    filename = f"{md5_hash}.{extension}"
    return f"/data/files/{bucket}/{filename}"

# 示例：MD5 为 a1b2c3d4... 的 PDF 文件
```
